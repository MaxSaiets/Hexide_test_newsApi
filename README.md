# News API (Test Task)

## Що реалізовано (Бекенд/API)

- **Новини**: Створення, редагування, видалення та перегляд новин по slug.
- **Блокова структура контенту**: Тіло новини складається з різних видів блоків (просто текст, картинка, текст з картинкою). Їх позицію можна змінювати.
- **Аутентифікація**: Реєстрація та авторизація реалізована через Laravel Sanctum (по токенах).
- **Swagger документація**: Для тестування апішок підключено пакет L5-Swagger. Сторінка з докою доступна за адресою `/api/documentation`.
- **Сортування та фільтрація**: Виведення можна фільтрувати (по пошуку) та показуються тільки ті, де статус `is_published = 1`.

## Фронтенд (Blade + Tailwind)

Створено кілька простих сторінок на Blade використовуючі шаблони із tailwind. **Вони зроблені виключно для того, щоб можна було швидко поклацати і перевірити**, як виводяться новини зі збірки блоків, і просто показати.
Що можна подивитись:

- Головна сторінка (`/`)
- Список новин з пошуком та пагінацією (`/news`)
- Перегляд конкретної новини з усіма її блоками (`/news/{slug}`)

## Як розгорнути локально

1. Клонувати репозиторій і перейти в папку:
   ```bash
   git clone https://github.com/MaxSaiets/Hexide_test_newsApi.git
   cd Hexide_test_newsApi
   ```
2. Встановити PHP-залежності:
   ```bash
   composer install
   ```
3. Скопіювати `.env.example` в `.env` і налаштувати підключення до БД (в мене стояла MySQL).
4. Згенерувати ключ додатку:
   ```bash
   php artisan key:generate
   ```
5. Запустити міграції:
   ```bash
   php artisan migrate
   ```
6. Наповнити базу тестовими даними (працюють Factory & Seeder):
   ```bash
   php artisan db:seed
   ```

   *Це створить тестових юзерів, новини та згенерує для них блоки з контентом, щоб було що потикати на сайті.*
7. (Обов'язково) Лінкувати сторедж для картинок:
   ```bash
   php artisan storage:link
   ```
8. Згенерувати документацію Swagger:
   ```bash
   php artisan l5-swagger:generate
   ```
9. Зібрати фронтенд для візуалу Blade:
   ```bash
   npm install
   npm run build
   ```
10. Запустити сервер:
    ```bash
    php artisan serve
    ```

## Архітектурні дрібниці

- Використано FormRequests (наприклад `StoreNewsBlockRequest`, `UpdateNewsRequest`) для валідації вхідних даних.
- API ресурси (JsonResource) для форматування відповідей, щоб не віддавати зайве.
- Картинки зберігаються локально в `storage/app/public` (через фасад Storage).
